# **三、Skills / Tool 设计（非常关键）**





在 Agent 系统里，**Skills 才是真正执行任务的模块**。



你可以设计一个：



**Quant Skills Registry**



示例：

```
skills = [
"generate_factor",
"evaluate_factor",
"backtest_strategy",
"store_factor",
"search_factor_db",
"generate_report"
]
```

每个 skill 对应一个函数。



例如：

```
def evaluate_factor(factor):
    ic = compute_ic(factor)
    icir = compute_icir(factor)
    return {"IC": ic, "ICIR": icir}
```

Agent 调用：

```
Tool: evaluate_factor
```

# **四、RAG系统（Factor Knowledge Base）**





RAG用于：

```
历史因子检索
避免重复研究
```

数据库结构：

```
factor_expression
IC
ICIR
Sharpe
decay
timestamp
```

向量化：

```
factor expression → embedding
```

流程：

```
new factor
↓
embedding
↓
vector search
↓
similar factor
```

这样 Agent 会知道：

```
这个因子已经研究过
```

```
Research memory
```

# **六、MCP（Model Context Protocol）**





MCP的作用：

```
统一模型与工具之间的通信
```

类似：

```
LLM ←→ Tools
```

MCP server可以提供：

```
factor_db_server
backtest_server
market_data_server
```

Agent调用：

```
call MCP tool: evaluate_factor
```

好处：



- 模块解耦
- 工具可扩展





很多AI系统现在用这种方式。



------





# **七、任务调度（Orchestrator）**





在多Agent系统中，需要一个：



**Task Orchestrator**



负责：

```
Agent之间通信
任务调度
失败重试
```

例如：

```
Planner Agent
↓
create tasks
↓
Task Queue
↓
Workers (Agents)
```

可以用：



- Python async
- Celery
- simple task queue
